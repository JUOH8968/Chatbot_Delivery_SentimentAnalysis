# 🚀 배달 음식 리뷰 감성 분석 시스템 향후 개선사항

현재 `my-chatbot` 프로토타입은 배달 도메인 데이터를 직접 파인튜닝(RoBERTa, 95% 정확도)하고 휴리스틱 다수결 판별 로직을 결합한 훌륭한 시스템입니다. 이를 상용화 수준(Production-Ready)의 실제 서비스로 발전시키기 위해 다음과 같은 3가지 기술적 구조 개선을 제안합니다.

---

## 1. ⚡ 모델 서빙(Model Serving) 최적화 및 경량화 아키텍처 도입

*   ''현재 상태 (AS-IS):'' 
    - `review.py` (FastAPI 앱) 내부에서 `AutoModelForSequenceClassification` 객체를 로드하여 인메모리로 API와 모델 추론을 동시에 처리하고 있어, 동시 접속자 증가 시 병목 현상(Bottleneck)과 메모리 과부하가 발생할 수 있습니다.
*   ''개선 목표 (TO-BE):'' 
    - 학습된 PyTorch 모델 포맷을 ''ONNX(Open Neural Network Exchange)''나 ''TensorRT''로 변환하여 추론(Inference) 속도와 메모리 효율성을 극대화합니다.
    - 웹 API 서버(FastAPI)와 ''추론 전용 서버(Triton Inference Server, BentoML 등)''를 분리(Decoupling)하여 대규모 트래픽을 처리하는 마이크로서비스 아키텍처(MSA)를 구축합니다.

## 2. 🧠 ABSA(속성 기반 감성 분석) 모델 도입을 통한 판별 고도화

*   ''현재 상태 (AS-IS):'' 
    - AI 감정 분석 외에 세부 3대 요소(맛, 가격/양, 배달) 판별을 위해, '맛', '벌레', '신속' 등 미리 정의해 놓은 단어사전(Keyword List) 기반의 휴리스틱 매칭 로직을 적용 중입니다. 신조어나 은어가 방대하게 늘어날 경우 파악에 한계가 있습니다.
*   ''개선 목표 (TO-BE):'' 
    - 단순 문장 긍/부정을 넘어, 문장 속 특정 속성(Aspect)에 대한 감정을 뽑아내는 ''ABSA (Aspect-Based Sentiment Analysis) 딥러닝 기법''을 추가로 도입합니다.
    - 하드코딩된 규칙(Rule) 의존도를 낮추고, 딥러닝 모델 스스로가 '음식이 식었다', '머리카락이 나왔다' 같은 복합적인 맥락을 추출해 내는 완전 자동화된 하이브리드 판별 시스템으로 진화시킵니다.

## 3. 🔄 데이터 파이프라인 및 실시간 MLOps 지속 학습 체계 구축

*   ''현재 상태 (AS-IS):'' 
    - 사용자가 리뷰 텍스트를 입력하면 분석 결과값이 1회성 API JSON 응답으로 반환되고 폐기되는 구조로, 예측 데이터가 축적되지 않습니다.
*   ''개선 목표 (TO-BE):'' 
    - 사용자의 텍스트 모델 예측 결과(긍정/부정, 카테고리 정보)를 ''PostgreSQL이나 ELK Stack(Elasticsearch) 등 DB에 실시간 적재''합니다.
    - 축적된 데이터를 모아 점주들이 식당 평판을 한눈에 볼 수 있도록 ''시각화 대시보드(BI 툴)''를 연동하며, 정기적으로 최신 신조어를 포함한 리뷰 데이터를 Airflow 같은 MLOps 파이프라인 수집기로 모아 ''RoBERTa 모델을 재학습(Continuous Training)하는 선순환 MLOps 생태계''를 마련합니다.
